De Superhero die ik had gekozen is Black Adam. De reden waarom dat ik Black Adam had gekozen is omdat ik en mijn familie de recentste film van hem hadden gezien genaamd Black AdamHet maken van deze website was niet zo moeilijk. Eerst moesten we een header maken voor de introductie (Superhero Banner). Vervolgens had ik een korte overzicht geschreven over de superhero met een foto boven de tekst.

Dan had ik de krachten geschreven van de superhero. De krachten had geschreven in een lijst. Omdat mijn lijst niet kwantitatief is had ik met de code ul gewerkt om dan met li een rij te maken. Elke kracht heeft zijn eigen li met een korte beschrijving van waar de alfabetische gerangschikte letter van Black Adam zijn kracht komt.

Daarna had moest ik 4 getuigen kiezen en hoe zij de superhero zien in hun visie en wat ze met hebben meegemaakt. De getuigen die ik had gekozen waren de 4 protagonisten van de film Black Adam(Doctor Fate/Kent Nelson, Carter Hall/Hawkman, Adrianna Tomaz, Amon). Het laatste dat er moest gebeuren was een Footer schrijven. In de Footer zitten alle bronnen van foto's die ik had gebruikt bij deze website. Geen bronvermelding is plagiaat en absoluut niet toegestaan.

Een deftige website is geen website zonder opmaak en daar komt CSS(Cascading Style Sheets) van toepassing. Bij CSS moet je rekening houden met de 'Composition, Utilities, Blocks & Exceptions' of de 'CUBE' regel. De macro elementen waren niet zo moeilijk hier om op te maken maar het waren vooral de micro-elementen omdat ze meestal in Composition en ook Utilities staan. Je kan zien aan mijn website dat ik alles centraal staat omdat ik bij mijn body de center waarde had gebruikt. De opmaak van mijn site had ik aangepast aan de kleuren van mijn superhero(zwart, goud en grijs). Mijn table elementen had ik op 100% width gezet zodat ik horizontale verlenging kon vermijden.

De micro-elementen waren moeilijker omdat het specifieke aanpassing zijn die je aan 1 code toevoegt. Dit valt onder 'Blocks'. Je kan dat bij verschillende elementen de text-breedte(font-size), text-stijl(font-family) en textkleur anders zijn. Ook bijna elke foto heeft zijn eigen width.Exceptions bij mij staat leeg omdat Exceptions regel iets is dat je zo weinig gaat gebruiken en zelfs kan vermijden te gebruiken.

De navigatie balk en de 'return' knop rechtsonder waren wel makkelijk te schrijven. Om zo'n elementen te maken moet je verwijzing maken. Dat doe je met a en dan een referentie met href direct erachter te schrijven. Let op: verwijzingen werken alleen met id's en niet met classes!!! Het verschil tussen die 2 is dat een id een specifieke verwijzing is en een class een groep is van elementen(bv: id="appel3" class='appels') Om naar een titel te verwijzen moet je binnen de nav element de a href met daarin de id van de titel. Dit is hoe bijna alle navigatie balken werken. Bij de return knop moest ik geen nav element gebruiken want dat is niet nodig. Alleen een a href met de id van de navigatie balk of een ander id dat jouw naar de top van de pagina brengt. Niet vergeten om in CSS de position van de return knop op fixed zodat het op je scherm blijft ongeacht als je scrollt zoals ik had gedaan(het moet niet maar het kan handig zijn).

Bij het maken van een website moet je altijd rekening houden met 'responsive design'. Responsive design is eigenlijk uw website opmaken zodat het even goed werkt op verschillende mediatypes (gsm, computer, tablets, ...). Om dit toe te passen moet je in CSS 'mediaqueries' gebruiken. Een mediaquery is een specifieke conditie dat een website moet aan voldoen om dat stukje opmaak toe te passen. Het is altijd best dat je mediaqueries helemaal beneden schrijft want CSS leest elementen van boven naar beneden. Hier onder vindt je een voorbeeld van een mediaquery. (Voorbeeld foto van mediaquery toevoegen) Bij mijn website moest ik geen mediaquery schrijven omdat mijn website even goed werkt op gsm en computer sinds dat alleen die 2 mediatypes gevraagd werden.
